# 数据结构-树

## 第 1 章 树结构概述与历史演进

### 1.1 树结构的起源与发展脉络

树结构的概念**起源于 19 世纪的图论研究**，早在 1847 年，G.R. 基尔霍夫在建立电网络理论时就提出了 "连通图"、"树"、"支撑树" 等基本概念。1857 年，A. 凯莱在研究饱和碳氢化合物同分异构体数目时独立地提出了树的概念，他把这类化合物的计数问题抽象为某类树的计数问题。

然而，树结构真正在计算机科学中得到广泛应用是在 20 世纪 50-60 年代。**1960 年随着计算机存储技术的发展，二叉树被正式引入数据结构领域**。这一时期，计算机科学家们开始系统研究树形结构在数据存储和检索中的应用价值。1960 年代初，著名计算机科学家艾兹赫尔・戴克斯特拉（Edsger Dijkstra）提出了基于二叉树的堆（Heap）数据结构，为后续的优先队列和排序算法奠定了基础。

树结构的发展可以划分为几个重要阶段：**早期探索阶段**（1950-1960 年代）主要关注基本概念的建立和简单应用；**算法优化阶段**（1960-1970 年代）重点解决平衡和效率问题；**工程应用阶段**（1970 年代至今）则专注于在数据库、文件系统等实际场景中的优化实现。

### 1.2 树结构解决的核心问题

树结构的出现主要是为了解决传统线性数据结构在**数据存储和检索效率**方面的瓶颈。早期的线性结构（如数组、链表）在面对大量数据时，查找操作的时间复杂度通常为 O (n)，这在数据规模增大时效率极低。

**二叉树的发明直接回应了计算机对 "比较" 和 "二分" 的需求**。通过将数据组织成二叉树结构，左子树存储较小的键，右子树存储较大的键，这种设计使得查找操作的时间复杂度降低到理想情况下的 O (log n)。这一突破为后续所有树形数据结构的发展奠定了理论基础。

随着应用场景的扩展，树结构需要解决更多复杂问题：**平衡问题**（避免退化为链表）、**磁盘 I/O 优化**（减少磁盘访问次数）、**字符串处理**（前缀匹配和检索）、**优先级管理**（高效获取最值元素）等。每种特定的树结构都是为解决某一具体性能瓶颈而设计的，体现了数据结构设计中 "问题驱动" 的核心思想。

### 1.3 树结构在数据结构体系中的地位

树结构在数据结构体系中占据着**承上启下的关键地位**。它既是线性结构的自然延伸，又是图结构的简化形式。树结构的层次特性使其成为描述具有分级特征数据的理想工具，广泛应用于操作系统目录结构、企业组织架构、XML 文档解析等场景。

从算法演进的角度看，树结构的发展推动了整个算法理论的进步。二叉搜索树的提出标志着**从线性查找向非线性查找的重要转变**；平衡二叉树的出现解决了最坏情况性能问题；B 树系列的发明则实现了内存与外存的高效交互。这些创新不仅解决了具体的技术问题，更为后续的算法设计提供了重要思路。

现代计算机科学中，树结构已经成为许多核心技术的基础。在数据库系统中，B + 树是标准的索引结构；在编译技术中，语法树是代码解析的核心；在人工智能领域，决策树是机器学习的重要算法；在网络路由中，最短路径树是路由算法的基础。可以说，**树结构是连接理论与实践的重要桥梁**，是每个计算机科学学习者必须掌握的核心内容。

## 第 2 章 二叉树基础

### 2.1 二叉树的概念与抽象逻辑

#### 2.1.1 二叉树的定义与特征

二叉树是一种**每个节点最多拥有两棵子树，且子树有明确左右次序的分层数据结构**。它是树结构家族中应用最广泛的分支，其严格的结构性定义为后续的算法设计提供了坚实基础。

二叉树的递归定义如下：二叉树要么是空树，要么由一个根节点和两棵互不相交的左子树、右子树组成，这两棵子树本身也分别是二叉树。与普通树相比，二叉树具有两个**本质特征**：

|对比维度|普通树|二叉树|
|---|---|---|
|节点最大度|无限制|最多为 2|
|子树次序|无左右区分|严格区分左右，次序不可颠倒|
这种严格的结构限制带来了许多便利。首先，二叉树的节点关系可以通过简单的数学公式表示；其次，二叉树的遍历算法具有统一的实现方式；最后，二叉树的各种操作都有明确的执行路径。

#### 2.1.2 二叉树的基本性质

二叉树具有一系列重要性质，这些性质是理解和应用二叉树的基础：

**性质 1（层级节点数）**：二叉树第 i 层（i≥1）至多有 2^(i-1) 个节点。这一性质由二叉树每个节点最多 2 个子节点的特性推导而来，用于计算指定层级的最大存储容量。

**性质 2（总节点数）**：深度为 k 的二叉树至多有 2^k - 1 个节点。该性质是层级节点数性质的求和结果，对应满二叉树的总节点数。

**性质 3（叶子节点与度为 2 节点的关系）**：对任意二叉树，叶子节点数 n0、度为 2 的节点数 n2 满足**n0 = n2 + 1**。这一性质通过总节点数与总边数的等式推导而来，是解决二叉树相关计算问题的关键公式。

#### 2.1.3 特殊二叉树类型

**满二叉树**是深度为 k 且恰好有 2^k - 1 个节点的二叉树，所有层级均被节点填满，不存在空缺位置。满二叉树的特点是所有叶子节点均在最底层，所有非叶子节点的度均为 2。

**完全二叉树**是深度为 k、有 n 个节点的二叉树，其节点按层序编号后，与同深度满二叉树编号 1 至 n 的节点位置完全一一对应，仅允许最底层右侧存在节点空缺。完全二叉树在数组存储方式下具有特殊优势，可以通过简单的下标计算实现父子节点的快速定位。

### 2.2 二叉树的存储方式

#### 2.2.1 顺序存储结构

顺序存储结构是指用一组地址连续的存储单元依次自上而下、自左至右存储完全二叉树上的结点元素。这种存储方式的核心思想是利用完全二叉树的编号规则，将节点按层序存储在数组中。

在顺序存储中，**节点的父子关系可以通过简单的数学公式计算**：

- 节点 i 的左孩子：2i + 1

- 节点 i 的右孩子：2i + 2

- 节点 i 的父节点：(i - 1) / 2

这种存储方式的**主要优势**在于：

1. 存储紧凑：对完全二叉树而言，存储空间利用率 100%，无指针开销

2. 访问速度快：通过数组下标直接访问，时间复杂度为 O (1)

3. 实现简单：无需复杂的指针操作

然而，顺序存储也存在明显的**局限性**：

1. 空间浪费严重：对于非完全二叉树，必须填充 "虚节点" 以维持下标规则

2. 动态操作困难：插入和删除节点需要移动大量元素

3. 空间预分配：需要预先分配足够的数组空间

**适用场景**：完全二叉树和满二叉树采用顺序存储比较合适，因为它们的结点序号可以唯一地反映结点之间的逻辑关系。在实际项目中，例如开发一个简单的文件系统目录树时，如果二叉树接近完全二叉树，顺序存储能高效利用内存。

#### 2.2.2 链式存储结构

链式存储结构是指用链表来表示一棵二叉树，即用链来指示元素的逻辑关系。链表中每个节点通常包含三个域：数据域（data）、左指针域（lchild）和右指针域（rchild）。

链式存储的**典型节点结构**：

```Plain Text

struct TreeNode {

&#x20;   int data;

&#x20;   TreeNode\* left;

&#x20;   TreeNode\* right;

&#x20;   TreeNode(int val) : data(val), left(nullptr), right(nullptr) {}

};
```

链式存储的**主要优势**：

1. 空间利用率高：只存储实际存在的节点，无需为不存在的节点预留空间

2. 动态操作方便：插入和删除操作只需修改相关指针，无需移动大量元素

3. 适应任意二叉树：可以表示任何形态的二叉树

链式存储的**主要缺点**：

1. 指针开销：每个节点需要额外的指针存储空间

2. 访问速度慢：需要通过指针逐层遍历，不能直接定位

3. 实现复杂：需要处理复杂的指针操作和内存管理

为了方便访问父节点，还可以设计**三叉链表结构**，增加一个 parent 指针指向父节点。这种结构既便于查找孩子节点，又便于查找双亲节点，但相对于二叉链表而言增加了空间开销。

#### 2.2.3 存储方式的选择策略

在选择二叉树的存储方式时，需要综合考虑以下因素：

**空间效率**：如果二叉树接近完全二叉树，顺序存储的空间利用率更高；如果二叉树形态复杂或节点分布稀疏，链式存储更合适。

**时间效率**：对于频繁的查找操作，顺序存储的 O (1) 访问速度具有明显优势；对于频繁的插入和删除操作，链式存储的 O (1) 指针修改更高效。

**实现复杂度**：顺序存储的实现相对简单，适合初学者；链式存储需要处理复杂的指针逻辑，但灵活性更高。

下表总结了两种存储方式的对比：

|存储方式|优点|缺点|适用场景|
|---|---|---|---|
|顺序存储|存储紧凑、访问快速、实现简单|空间浪费、动态操作困难|完全二叉树、满二叉树|
|链式存储|空间利用率高、动态操作方便|指针开销、访问较慢|任意形态二叉树|
### 2.3 二叉树的遍历算法

#### 2.3.1 遍历算法的基本概念

二叉树的遍历是指按照固定规则访问二叉树所有节点且每个节点仅访问一次的操作。根据访问根节点的时机不同，二叉树有三种基本遍历方式，它们都遵循 \*\* 根（N）、左子树（L）、右子树（R）\*\* 的相对顺序：

1. **前序遍历（NLR）**：根节点→左子树→右子树

2. **中序遍历（LNR）**：左子树→根节点→右子树

3. **后序遍历（LRN）**：左子树→右子树→根节点

此外，还有一种基于层次的遍历方式：

4. **层序遍历**：从上到下、从左到右逐层访问节点

这些遍历算法是二叉树所有操作的基础，在实际应用中具有广泛用途：

- 前序遍历：适合需要 "先处理根节点" 的场景，如复制二叉树、获取树的前缀序列

- 中序遍历：对于二叉搜索树，中序遍历输出有序序列

- 后序遍历：常用于二叉树的删除操作和计算表达式值

- 层序遍历：用于计算树的深度、宽度等属性

#### 2.3.2 递归实现方式

递归是实现二叉树遍历最自然的方式，代码简洁且易于理解。以 C++ 实现为例：

**前序遍历递归实现**：

```Plain Text

void preorderTraversal(TreeNode\* root) {

&#x20;   if (root == nullptr) return;

&#x20;   cout << root->val << " ";  // 访问根节点

&#x20;   preorderTraversal(root->left);  // 递归遍历左子树

&#x20;   preorderTraversal(root->right);  // 递归遍历右子树

}
```

**中序遍历递归实现**：

```Plain Text

void inorderTraversal(TreeNode\* root) {

&#x20;   if (root == nullptr) return;

&#x20;   inorderTraversal(root->left);  // 递归遍历左子树

&#x20;   cout << root->val << " ";  // 访问根节点

&#x20;   inorderTraversal(root->right);  // 递归遍历右子树

}
```

**后序遍历递归实现**：

```Plain Text

void postorderTraversal(TreeNode\* root) {

&#x20;   if (root == nullptr) return;

&#x20;   postorderTraversal(root->left);  // 递归遍历左子树

&#x20;   postorderTraversal(root->right);  // 递归遍历右子树

&#x20;   cout << root->val << " ";  // 访问根节点

}
```

递归实现的优点是逻辑清晰、代码简洁，但也存在一些缺点：

1. 递归深度可能过大，导致栈溢出

2. 函数调用开销较大

3. 不适合某些特定的优化需求

#### 2.3.3 迭代实现方式

迭代实现通过显式使用栈结构模拟递归过程，可以避免递归的一些缺点。以下是三种遍历的迭代实现：

**前序遍历迭代实现**：

```Plain Text

vector\<int> preorderTraversal(TreeNode\* root) {

&#x20;   stack\<TreeNode\*> S;

&#x20;   vector\<int> result;

&#x20;   TreeNode\* current = root;

&#x20;  &#x20;

&#x20;   while (current || !S.empty()) {

&#x20;       while (current) {

&#x20;           result.push\_back(current->val);  // 访问当前节点

&#x20;           S.push(current->right);  // 右子树入栈

&#x20;           current = current->left;  // 移动到左子树

&#x20;       }

&#x20;       current = S.top();  // 取出栈顶元素

&#x20;       S.pop();

&#x20;   }

&#x20;   return result;

}
```

**中序遍历迭代实现**：

```Plain Text

vector\<int> inorderTraversal(TreeNode\* root) {

&#x20;   stack\<TreeNode\*> S;

&#x20;   vector\<int> result;

&#x20;   TreeNode\* current = root;

&#x20;  &#x20;

&#x20;   while (current || !S.empty()) {

&#x20;       while (current) {

&#x20;           S.push(current);  // 当前节点入栈

&#x20;           current = current->left;  // 移动到左子树

&#x20;       }

&#x20;       current = S.top();  // 取出栈顶元素

&#x20;       S.pop();

&#x20;       result.push\_back(current->val);  // 访问当前节点

&#x20;       current = current->right;  // 移动到右子树

&#x20;   }

&#x20;   return result;

}
```

**后序遍历迭代实现**：

```Plain Text

vector\<int> postorderTraversal(TreeNode\* root) {

&#x20;   stack\<TreeNode\*> S;

&#x20;   vector\<int> result;

&#x20;   TreeNode\* current = root;

&#x20;  &#x20;

&#x20;   while (current || !S.empty()) {

&#x20;       while (current) {

&#x20;           result.push\_back(current->val);  // 访问当前节点

&#x20;           S.push(current->left);  // 左子树入栈

&#x20;           current = current->right;  // 移动到右子树

&#x20;       }

&#x20;       current = S.top();  // 取出栈顶元素

&#x20;       S.pop();

&#x20;   }

&#x20;   reverse(result.begin(), result.end());  // 反转结果

&#x20;   return result;

}
```

迭代实现的核心思想是：

1. 使用栈保存待处理的节点

2. 通过循环模拟递归调用过程

3. 前序遍历：先访问节点，再将右子树入栈，最后处理左子树

4. 中序遍历：先将所有左子树节点入栈，然后依次取出访问并处理右子树

5. 后序遍历：通过反向处理，最后反转结果得到正确顺序

#### 2.3.4 层序遍历算法

层序遍历是从上到下、从左到右依次访问二叉树的所有节点。算法使用队列来实现：

**层序遍历实现**：

```Plain Text

vector\<vector\<int>> levelOrder(TreeNode\* root) {

&#x20;   vector\<vector\<int>> result;

&#x20;   if (root == nullptr) return result;

&#x20;  &#x20;

&#x20;   queue\<TreeNode\*> Q;

&#x20;   Q.push(root);

&#x20;  &#x20;

&#x20;   while (!Q.empty()) {

&#x20;       int levelSize = Q.size();

&#x20;       vector\<int> level;

&#x20;      &#x20;

&#x20;       for (int i = 0; i < levelSize; i++) {

&#x20;           TreeNode\* current = Q.front();

&#x20;           Q.pop();

&#x20;           level.push\_back(current->val);

&#x20;          &#x20;

&#x20;           if (current->left) Q.push(current->left);

&#x20;           if (current->right) Q.push(current->right);

&#x20;       }

&#x20;       result.push\_back(level);

&#x20;   }

&#x20;   return result;

}
```

层序遍历的实现步骤：

1. 初始化队列，将根节点入队

2. 循环处理队列中的节点

3. 每次处理一层节点，记录该层的所有节点值

4. 将下一层的节点依次入队

5. 直到队列为空，处理完成

这种实现方式的时间复杂度为 O (n)，其中 n 是二叉树的节点总数，因为每个节点被访问一次。

#### 2.3.5 遍历算法的复杂度分析

所有遍历算法的时间复杂度均为**O(n)**，其中 n 是二叉树的节点总数。这是因为每个节点都必须被访问且仅访问一次。

空间复杂度方面：

- 递归实现：最坏情况下（单支树）空间复杂度为 O (n)，即递归栈的深度

- 迭代实现：使用栈或队列，最坏情况下空间复杂度也为 O (n)

- 层序遍历：使用队列，空间复杂度为 O (n)，用于存储一层的所有节点

## 第 3 章 二叉搜索树

### 3.1 二叉搜索树的概念与性质

#### 3.1.1 二叉搜索树的定义

二叉搜索树（Binary Search Tree, BST）是计算机科学中最重要的数据结构之一，由 P.F. Windley、A.D. Booth、A.J.T. Colin 和 T.N. Hibbard 在 1960 年独立提出。

二叉搜索树的定义：二叉搜索树是空树，或满足以下性质的二叉树：

- 左子树所有节点值小于根节点值

- 右子树所有节点值大于根节点值

- 左右子树也分别为二叉搜索树

这种递归定义确保了二叉搜索树的**有序性**，使得查找、插入、删除等操作都可以在对数时间内完成（在平衡情况下）。

#### 3.1.2 二叉搜索树的核心性质

二叉搜索树具有以下重要性质：

1. **中序遍历有序性**：二叉搜索树的中序遍历结果为严格递增的有序序列。这是二叉搜索树最核心的优势，直接支撑数据库的有序查询需求。

2. **查找性质**：从根节点开始，目标值小于根则查找左子树，大于根则查找右子树，相等则命中。这种二分查找的思想使得查找效率大大提高。

3. **递归结构**：每个子树也都是二叉搜索树，这种递归性质为算法设计提供了便利。

4. **唯一性质**：通常不允许重复节点，若插入值已存在则不插入或更新。

#### 3.1.3 二叉搜索树的应用场景

二叉搜索树在实际应用中具有广泛用途：

- **数据库索引**：B + 树是多路平衡的二叉搜索树扩展，成为关系型数据库的标准索引结构

- **符号表**：用于实现字典、映射等数据结构

- **编译器**：用于管理变量符号表

- **文件系统**：用于目录结构的组织和查找

### 3.2 二叉搜索树的操作实现

#### 3.2.1 查找操作

二叉搜索树的查找操作是其核心功能之一。查找过程从根节点开始，逐层比较：

**查找算法实现**：

```Plain Text

TreeNode\* searchBST(TreeNode\* root, int val) {

&#x20;   if (root == nullptr || root->val == val) {

&#x20;       return root;

&#x20;   }

&#x20;   if (val < root->val) {

&#x20;       return searchBST(root->left, val);  // 递归查找左子树

&#x20;   } else {

&#x20;       return searchBST(root->right, val);  // 递归查找右子树

&#x20;   }

}
```

查找操作的步骤：

1. 如果当前节点为空或找到目标值，返回当前节点

2. 如果目标值小于当前节点值，递归查找左子树

3. 如果目标值大于当前节点值，递归查找右子树

**迭代实现**：

```Plain Text

TreeNode\* searchBSTIterative(TreeNode\* root, int val) {

&#x20;   while (root != nullptr) {

&#x20;       if (val == root->val) {

&#x20;           return root;

&#x20;       } else if (val < root->val) {

&#x20;           root = root->left;

&#x20;       } else {

&#x20;           root = root->right;

&#x20;       }

&#x20;   }

&#x20;   return nullptr;

}
```

#### 3.2.2 插入操作

插入操作需要保持二叉搜索树的有序性。插入过程从根节点开始，沿适当路径找到插入位置：

**插入算法实现**：

```Plain Text

TreeNode\* insertIntoBST(TreeNode\* root, int val) {

&#x20;   if (root == nullptr) {

&#x20;       return new TreeNode(val);  // 创建新节点作为根

&#x20;   }

&#x20;  &#x20;

&#x20;   if (val < root->val) {

&#x20;       root->left = insertIntoBST(root->left, val);  // 递归插入左子树

&#x20;   } else {

&#x20;       root->right = insertIntoBST(root->right, val);  // 递归插入右子树

&#x20;   }

&#x20;   return root;

}
```

插入操作的详细步骤：

1. 若树为空，直接创建新节点作为根节点

2. 比较插入值与当前节点值：

- 小于当前节点值则递归插入左子树

- 大于当前节点值则递归插入右子树

- 等于当前节点值则不插入（BST 不允许重复节点）

**迭代实现**：

```Plain Text

TreeNode\* insertIntoBSTIterative(TreeNode\* root, int val) {

&#x20;   if (root == nullptr) return new TreeNode(val);

&#x20;  &#x20;

&#x20;   TreeNode\* current = root;

&#x20;   TreeNode\* parent = nullptr;

&#x20;  &#x20;

&#x20;   while (current != nullptr) {

&#x20;       parent = current;

&#x20;       if (val < current->val) {

&#x20;           current = current->left;

&#x20;       } else {

&#x20;           current = current->right;

&#x20;       }

&#x20;   }

&#x20;  &#x20;

&#x20;   if (val < parent->val) {

&#x20;       parent->left = new TreeNode(val);

&#x20;   } else {

&#x20;       parent->right = new TreeNode(val);

&#x20;   }

&#x20;   return root;

}
```

#### 3.2.3 删除操作

删除操作是二叉搜索树中最复杂的操作，需要考虑三种情况：

1. **删除叶子节点**：直接删除，无需调整

2. **删除只有一个子节点的节点**：用子节点替换当前节点

3. **删除有两个子节点的节点**：找到中序后继（右子树的最小值），用后继值覆盖当前节点，然后删除后继节点

**删除算法实现**：

```Plain Text

TreeNode\* deleteNode(TreeNode\* root, int key) {

&#x20;   if (root == nullptr) return root;

&#x20;  &#x20;

&#x20;   if (key < root->val) {

&#x20;       root->left = deleteNode(root->left, key);  // 递归删除左子树

&#x20;   } else if (key > root->val) {

&#x20;       root->right = deleteNode(root->right, key);  // 递归删除右子树

&#x20;   } else {

&#x20;       // 找到要删除的节点

&#x20;       if (root->left == nullptr) {

&#x20;           // 只有右子树或无子女

&#x20;           TreeNode\* temp = root->right;

&#x20;           delete root;

&#x20;           return temp;

&#x20;       } else if (root->right == nullptr) {

&#x20;           // 只有左子树

&#x20;           TreeNode\* temp = root->left;

&#x20;           delete root;

&#x20;           return temp;

&#x20;       }

&#x20;      &#x20;

&#x20;       // 有两个子节点：找到中序后继

&#x20;       TreeNode\* temp = findMin(root->right);

&#x20;       root->val = temp->val;  // 用后继值覆盖

&#x20;       root->right = deleteNode(root->right, temp->val);  // 删除后继

&#x20;   }

&#x20;   return root;

}

TreeNode\* findMin(TreeNode\* root) {

&#x20;   while (root->left != nullptr) {

&#x20;       root = root->left;

&#x20;   }

&#x20;   return root;

}
```

删除操作的详细步骤：

1. 查找要删除的节点

2. 根据子节点情况处理：

- 无左子树：用右子树替换当前节点

- 无右子树：用左子树替换当前节点

- 有两个子树：

- 步骤 1：找中序后继（右子树中的最小值）

- 步骤 2：用后继值覆盖被删节点

- 步骤 3：递归删除后继节点（此时后继节点必为叶子节点）

#### 3.2.4 中序遍历验证有序性

为了验证二叉树是否为二叉搜索树，可以通过中序遍历检查序列的有序性：

**验证 BST 实现**：

```Plain Text

bool isValidBST(TreeNode\* root) {

&#x20;   stack\<TreeNode\*> S;

&#x20;   TreeNode\* prev = nullptr;

&#x20;  &#x20;

&#x20;   while (root || !S.empty()) {

&#x20;       while (root) {

&#x20;           S.push(root);

&#x20;           root = root->left;

&#x20;       }

&#x20;      &#x20;

&#x20;       root = S.top();

&#x20;       S.pop();

&#x20;      &#x20;

&#x20;       if (prev != nullptr && root->val <= prev->val) {

&#x20;           return false;

&#x20;       }

&#x20;       prev = root;

&#x20;       root = root->right;

&#x20;   }

&#x20;   return true;

}
```

验证算法的思路：

1. 使用中序遍历的迭代方式

2. 记录前一个访问的节点

3. 检查当前节点值是否大于前一个节点值

4. 如果出现逆序，说明不是二叉搜索树

### 3.3 二叉搜索树的复杂度分析

#### 3.3.1 时间复杂度分析

二叉搜索树的时间复杂度取决于树的高度。在理想平衡状态下，树的高度为 O (log n)，但在最坏情况下，树可能退化为链表，高度为 O (n)。

**各种操作的时间复杂度**：

|操作|平均情况|最坏情况|
|---|---|---|
|查找|O(log n)|O(n)|
|插入|O(log n)|O(n)|
|删除|O(log n)|O(n)|
|遍历|O(n)|O(n)|
**详细分析**：

- 查找操作：每次比较后排除一半节点，理想情况下时间复杂度为 O (log n)

- 插入操作：找到插入位置需要 O (log n) 时间，创建节点为 O (1)

- 删除操作：查找节点需要 O (log n) 时间，调整树结构最多需要 O (log n) 时间

- 最坏情况：当树退化为链表时（如有序插入），所有操作的时间复杂度均为 O (n)

#### 3.3.2 空间复杂度分析

二叉搜索树的空间复杂度为**O(n)**，用于存储 n 个节点。每个节点需要存储数据和两个指针（左、右子节点）。

在递归实现中，递归栈的深度在最坏情况下为 O (n)，因此总的空间复杂度为 O (n)。

#### 3.3.3 性能优化考虑

二叉搜索树的性能瓶颈主要在于可能退化为链表。为了解决这一问题，出现了各种平衡二叉树，如 AVL 树、红黑树等，它们通过自动调整树的结构来保持平衡，确保所有操作的时间复杂度稳定在 O (log n)。

## 第 4 章 平衡二叉树

### 4.1 AVL 树的原理与实现

#### 4.1.1 AVL 树的概念与背景

AVL 树是最早被发明的自平衡二叉查找树，得名于其发明者 G.M. Adelson-Velsky 和 E.M. Landis（1962 年发表相关论文）。它的出现是为了解决二叉搜索树在最坏情况下退化为链表的问题，确保查找、插入、删除等操作的时间复杂度稳定在 O (log n)。

AVL 树的定义：AVL 树是一种自平衡的二叉搜索树，任何节点的两个子树的高度差不超过 1。这个高度差被称为**平衡因子**，平衡因子的取值只能是 - 1、0 或 1。

#### 4.1.2 AVL 树的平衡因子

AVL 树的每个节点都需要维护一个平衡因子（balance factor），定义为：

```Plain Text

平衡因子 = 左子树高度 - 右子树高度
```

平衡因子的可能取值：

- **-1**：右子树比左子树高 1 层

- **0**：左右子树高度相等

- **1**：左子树比右子树高 1 层

当某个节点的平衡因子绝对值超过 1 时，树就失去了平衡，需要进行旋转操作来恢复平衡。

#### 4.1.3 旋转操作详解

AVL 树通过四种旋转操作来恢复平衡：**左单旋**、**右单旋**、**左右双旋**、**右左双旋**。每种旋转操作都保持二叉搜索树的性质不变。

**1. 左单旋（Left Rotation）**

左单旋用于处理 RR 型失衡（右子树的右子树插入导致失衡）。

左单旋的步骤：

1. 让 k2（失衡节点的右孩子）原本指向 k1 的指针现在指向 k1 内侧的节点

2. 让 k1 原本指向内侧的指针现在指向 k2

3. 让原来指向 k2 的指针现在改为指向 k1，并更新各节点的高度

左单旋的详细实现：

```Plain Text

Node\* rotateLeft(Node\* parent) {

&#x20;   Node\* subR = parent->right;  // 取parent的右孩子

&#x20;   Node\* subRL = subR->left;    // 取subR的左孩子

&#x20;  &#x20;

&#x20;   // 步骤1：建立subR和parent之间的关系

&#x20;   parent->parent = subR;

&#x20;   subR->left = parent;

&#x20;  &#x20;

&#x20;   // 步骤2：建立parent和subRL之间的关系

&#x20;   parent->right = subRL;

&#x20;   if (subRL) subRL->parent = parent;

&#x20;  &#x20;

&#x20;   // 步骤3：建立parentParent和subR之间的关系

&#x20;   if (parent == root) {

&#x20;       root = subR;

&#x20;       subR->parent = nullptr;

&#x20;   } else {

&#x20;       if (parent == parent->parent->left) {

&#x20;           parent->parent->left = subR;

&#x20;       } else {

&#x20;           parent->parent->right = subR;

&#x20;       }

&#x20;       subR->parent = parent->parent;

&#x20;   }

&#x20;  &#x20;

&#x20;   // 更新平衡因子

&#x20;   subR->bf = parent->bf = 0;

&#x20;   return subR;

}
```

**2. 右单旋（Right Rotation）**

右单旋用于处理 LL 型失衡（左子树的左子树插入导致失衡）。

右单旋的步骤：

1. 让 k1（失衡节点的左孩子）原本指向 k2 的指针现在指向 k2 内侧的节点

2. 让 k2 原本指向内侧的指针现在指向 k1

3. 让原来指向 k1 的指针现在改为指向 k2，并更新各节点的高度

右单旋的实现：

```Plain Text

Node\* rotateRight(Node\* parent) {

&#x20;   Node\* subL = parent->left;   // 取parent的左孩子

&#x20;   Node\* subLR = subL->right;   // 取subL的右孩子

&#x20;  &#x20;

&#x20;   // 步骤1：建立subL和parent之间的关系

&#x20;   subL->right = parent;

&#x20;   parent->parent = subL;

&#x20;  &#x20;

&#x20;   // 步骤2：建立parent和subLR之间的关系

&#x20;   parent->left = subLR;

&#x20;   if (subLR) subLR->parent = parent;

&#x20;  &#x20;

&#x20;   // 步骤3：建立parentParent和subL之间的关系

&#x20;   if (parent == root) {

&#x20;       root = subL;

&#x20;       subL->parent = nullptr;

&#x20;   } else {

&#x20;       if (parent == parent->parent->left) {

&#x20;           parent->parent->left = subL;

&#x20;       } else {

&#x20;           parent->parent->right = subL;

&#x20;       }

&#x20;       subL->parent = parent->parent;

&#x20;   }

&#x20;  &#x20;

&#x20;   // 更新平衡因子

&#x20;   subL->bf = parent->bf = 0;

&#x20;   return subL;

}
```

**3. 左右双旋（Left-Right Rotation）**

左右双旋用于处理 LR 型失衡（左子树的右子树插入导致失衡）。处理步骤是先左旋后右旋。

左右双旋的实现：

```Plain Text

Node\* leftRightRotate(Node\* z) {

&#x20;   z->left = leftRotate(z->left);  // 第一步：左旋z的左子节点

&#x20;   return rightRotate(z);          // 第二步：右旋z

}
```

**4. 右左双旋（Right-Left Rotation）**

右左双旋用于处理 RL 型失衡（右子树的左子树插入导致失衡）。处理步骤是先右旋后左旋。

右左双旋的实现：

```Plain Text

Node\* rightLeftRotate(Node\* z) {

&#x20;   z->right = rightRotate(z->right);  // 第一步：右旋z的右子节点

&#x20;   return leftRotate(z);              // 第二步：左旋z

}
```

#### 4.1.4 旋转类型的判断

根据失衡节点及其子节点的平衡因子，可以确定旋转类型：

|失衡类型|失衡节点平衡因子|子节点平衡因子|旋转类型|
|---|---|---|---|
|LL 型|-2|-1|右单旋|
|LR 型|-2|1|左右双旋|
|RR 型|2|1|左单旋|
|RL 型|2|-1|右左双旋|
### 4.2 AVL 树的插入操作

#### 4.2.1 插入算法概述

AVL 树的插入操作分为两个阶段：

1. **标准 BST 插入**：将节点插入到适当位置，保持二叉搜索树性质

2. **平衡调整**：从插入节点开始，向上更新平衡因子，必要时进行旋转

#### 4.2.2 插入后的平衡调整

插入操作后的平衡调整过程：

1. 从插入节点开始，向上遍历到根节点

2. 更新每个祖先节点的平衡因子

3. 检查是否出现失衡（平衡因子绝对值超过 1）

4. 如果失衡，根据失衡类型进行相应的旋转

5. 旋转后，树恢复平衡，无需继续向上调整

**插入算法实现**：

```Plain Text

Node\* insertAVL(Node\* root, int val) {

&#x20;   if (root == nullptr) {

&#x20;       return new Node(val);  // 创建新节点

&#x20;   }

&#x20;  &#x20;

&#x20;   if (val < root->val) {

&#x20;       root->left = insertAVL(root->left, val);  // 递归插入左子树

&#x20;       root->bf--;  // 左子树插入，bf减1

&#x20;   } else if (val > root->val) {

&#x20;       root->right = insertAVL(root->right, val);  // 递归插入右子树

&#x20;       root->bf++;  // 右子树插入，bf加1

&#x20;   } else {

&#x20;       return root;  // 不允许重复

&#x20;   }

&#x20;  &#x20;

&#x20;   // 检查平衡并调整

&#x20;   if (abs(root->bf) > 1) {

&#x20;       balanceAVL(root);  // 进行平衡调整

&#x20;   }

&#x20;  &#x20;

&#x20;   return root;

}

void balanceAVL(Node\* root) {

&#x20;   if (root->bf == -2) {  // 右子树高

&#x20;       if (root->right->bf == 1) {  // RL型

&#x20;           rightLeftRotate(root);

&#x20;       } else {  // RR型

&#x20;           leftRotate(root);

&#x20;       }

&#x20;   } else if (root->bf == 2) {  // 左子树高

&#x20;       if (root->left->bf == -1) {  // LR型

&#x20;           leftRightRotate(root);

&#x20;       } else {  // LL型

&#x20;           rightRotate(root);

&#x20;       }

&#x20;   }

}
```

#### 4.2.3 插入过程的平衡因子更新

插入过程中平衡因子的更新规则：

- 如果在左子树插入节点，当前节点的平衡因子减 1

- 如果在右子树插入节点，当前节点的平衡因子加 1

平衡调整的关键在于：

1. 每次插入后，从插入点向上检查平衡状态

2. 一旦发现失衡，立即进行旋转调整

3. 旋转后树的高度恢复到插入前的状态，因此无需继续向上调整

### 4.3 AVL 树的删除操作

#### 4.3.1 删除算法概述

AVL 树的删除操作比插入更复杂，因为删除可能导致多个祖先节点失衡，需要从删除节点开始向上一直检查到根节点。

删除操作分为三个阶段：

1. **标准 BST 删除**：找到并删除目标节点

2. **平衡因子更新**：从删除节点的父节点开始，向上更新平衡因子

3. **平衡调整**：检查每个节点是否失衡，必要时进行旋转

#### 4.3.2 删除后的平衡调整

删除操作后的平衡调整与插入不同：

- 插入后只需调整到第一个平衡的节点

- 删除后可能需要一直调整到根节点，因为删除会降低树的高度

**删除算法实现**：

```Plain Text

Node\* deleteAVL(Node\* root, int val) {

&#x20;   if (root == nullptr) return root;

&#x20;  &#x20;

&#x20;   if (val < root->val) {

&#x20;       root->left = deleteAVL(root->left, val);

&#x20;       root->bf++;  // 左子树删除，bf加1

&#x20;   } else if (val > root->val) {

&#x20;       root->right = deleteAVL(root->right, val);

&#x20;       root->bf--;  // 右子树删除，bf减1

&#x20;   } else {

&#x20;       // 找到要删除的节点

&#x20;       if (root->left == nullptr) {

&#x20;           Node\* temp = root->right;

&#x20;           delete root;

&#x20;           return temp;

&#x20;       } else if (root->right == nullptr) {

&#x20;           Node\* temp = root->left;

&#x20;           delete root;

&#x20;           return temp;

&#x20;       } else {

&#x20;           Node\* temp = findMin(root->right);

&#x20;           root->val = temp->val;

&#x20;           root->right = deleteAVL(root->right, temp->val);

&#x20;           root->bf--;  // 右子树删除，bf减1

&#x20;       }

&#x20;   }

&#x20;  &#x20;

&#x20;   // 检查并调整平衡

&#x20;   if (abs(root->bf) > 1) {

&#x20;       balanceDeleteAVL(root);

&#x20;   }

&#x20;  &#x20;

&#x20;   return root;

}

void balanceDeleteAVL(Node\* root) {

&#x20;   if (root->bf == -2) {  // 右子树高

&#x20;       if (root->right->bf >= 0) {  // RR或RL型

&#x20;           if (root->right->bf == 1) {  // RL型

&#x20;               rightLeftRotate(root);

&#x20;           } else {  // RR型

&#x20;               leftRotate(root);

&#x20;           }

&#x20;       } else {  // 特殊情况：右子树的左子树为空

&#x20;           leftRotate(root);

&#x20;           root->bf = 1;  // 需要继续向上调整

&#x20;       }

&#x20;   } else if (root->bf == 2) {  // 左子树高

&#x20;       if (root->left->bf <= 0) {  // LL或LR型

&#x20;           if (root->left->bf == -1) {  // LR型

&#x20;               leftRightRotate(root);

&#x20;           } else {  // LL型

&#x20;               rightRotate(root);

&#x20;           }

&#x20;       } else {  // 特殊情况：左子树的右子树为空

&#x20;           rightRotate(root);

&#x20;           root->bf = -1;  // 需要继续向上调整

&#x20;       }

&#x20;   }

}
```

#### 4.3.3 删除操作的特殊情况

删除操作有一些特殊情况需要注意：

1. **删除后子树高度降低**：删除操作可能导致子树高度降低，需要继续向上检查

2. **旋转后仍需调整**：某些情况下，旋转后树的高度仍然降低，需要继续向上调整

### 4.4 AVL 树的复杂度分析

#### 4.4.1 时间复杂度分析

AVL 树的所有基本操作（查找、插入、删除）的时间复杂度均为**O(log n)**，这是因为 AVL 树始终保持平衡，树的高度被限制在 O (log n)。

**详细分析**：

- 查找操作：与 BST 相同，每次比较后排除一半节点，时间复杂度 O (log n)

- 插入操作：标准 BST 插入需要 O (log n) 时间，平衡调整最多需要 O (1) 次旋转，每次旋转为 O (1) 时间，总时间复杂度 O (log n)

- 删除操作：标准 BST 删除需要 O (log n) 时间，平衡调整可能需要 O (log n) 次旋转（因为可能需要一直调整到根节点），但每次旋转为 O (1)，总时间复杂度 O (log n)

#### 4.4.2 空间复杂度分析

AVL 树的空间复杂度为**O(n)**，每个节点需要存储：

- 数据值

- 左右子节点指针

- 平衡因子（通常用一个整数表示）

平衡因子的存储空间开销很小（通常只需要 1-2 位），因此空间复杂度与普通 BST 相当。

#### 4.4.3 性能特点

AVL 树的主要特点：

1. **严格平衡**：任何节点的左右子树高度差不超过 1

2. **查询效率高**：查找操作的时间复杂度稳定在 O (log n)

3. **插入 / 删除开销**：需要维护平衡，可能进行多次旋转

4. **适用场景**：适合查询密集型应用，如字典、符号表等

### 4.5 红黑树的原理与实现

#### 4.5.1 红黑树的概念与背景

红黑树是一种自平衡二叉查找树，由 Rudolf Bayer 于 1972 年发明（最初称为对称二叉 B 树），1978 年被 Leo J. Guibas 和 Robert Sedgewick 重新命名为 "红黑树"。红黑树比 AVL 树晚 10 年诞生，但由于其在随机数据插入性能上的优势，获得了更广泛的应用。

红黑树通过**颜色标记和一系列性质约束**来维持树的 "大致平衡"，不追求 AVL 树那样的绝对平衡，而是保证从根到叶子的最长路径不超过最短路径的两倍。

#### 4.5.2 红黑树的性质

红黑树的每个节点必须满足以下性质：

1. 每个节点非红即黑

2. 根节点总是黑色的

3. 每个叶子节点（NIL 节点）都是黑色的空节点

4. 如果节点是红色的，则它的子节点必须是黑色的（不会有连续的红色节点）

5. 从根节点到叶节点或空子节点的每条路径，必须包含相同数目的黑色节点（相同的黑色高度）

这些性质保证了红黑树的平衡：

- 最长路径（红黑交替）不超过最短路径（全黑）的两倍

- 树的高度不超过 2log (n+1)

#### 4.5.3 红黑树的节点结构

红黑树的节点结构需要额外的颜色信息：

#### 4.5.4 红黑树的旋转操作

红黑树的旋转操作与 AVL 树类似，但目的不同。红黑树的旋转主要用于调整颜色和保持红黑性质。

**左旋转实现**：

```Plain Text

void leftRotate(RBNode\*& root, RBNode\* x) {

&#x20;   RBNode\* y = x->right;  // 取x的右孩子

&#x20;   x->right = y->left;     // 将y的左子树变为x的右子树

&#x20;  &#x20;

&#x20;   if (y->left != nullptr) {

&#x20;       y->left->parent = x;

&#x20;   }

&#x20;  &#x20;

&#x20;   y->parent = x->parent;  // 连接y的父节点

&#x20;  &#x20;

&#x20;   if (x->parent == nullptr) {

&#x20;       root = y;

&#x20;   } else if (x == x->parent->left) {

&#x20;       x->parent->left = y;

&#x20;   } else {

&#x20;       x->parent->right = y;

&#x20;   }

&#x20;  &#x20;

&#x20;   y->left = x;            // 将x变为y的左孩子

&#x20;   x->parent = y;

}
```

**右旋转实现**：

```Plain Text

void rightRotate(RBNode\*& root, RBNode\* y) {

&#x20;  &#x20;

&#x20;   if (x->right != nullptr) {

&#x20;       x->right->parent = y;

&#x20;   }

&#x20;  &#x20;

&#x20;   x->parent = y->parent;  // 连接x的父节点

&#x20;  &#x20;

&#x20;   if (y->parent == nullptr) {

&#x20;   } else if (y == y->parent->left) {

&#x20;   } else {

&#x20;       y->parent->right = x;

&#x20;   }

&#x20;  &#x20;

}
```

#### 4.5.5 红黑树的插入操作

红黑树的插入操作分为两步：

1. **标准 BST 插入**：插入节点并设为红色

2. **颜色调整**：通过旋转和颜色变换保持红黑性质

插入后可能破坏的性质：

- 如果父节点是红色，可能出现连续的红色节点

**插入算法实现**：

```Plain Text

void insertRBTree(RBNode\*& root, RBNode\* z) {

&#x20;   RBNode\* y = nullptr;

&#x20;   RBNode\* x = root;

&#x20;  &#x20;

&#x20;   while (x != nullptr) {

&#x20;       y = x;

&#x20;       if (z->val < x->val) {

&#x20;           x = x->left;

&#x20;       } else {

&#x20;           x = x->right;

&#x20;       }

&#x20;   }

&#x20;  &#x20;

&#x20;   z->parent = y;

&#x20;  &#x20;

&#x20;   if (y == nullptr) {

&#x20;       root = z;  // 空树

&#x20;   } else if (z->val < y->val) {

&#x20;       y->left = z;

&#x20;   } else {

&#x20;       y->right = z;

&#x20;   }

&#x20;  &#x20;

&#x20;   z->color = RED;  // 新节点设为红色

&#x20;   insertFixup(root, z);  // 调整颜色

}

void insertFixup(RBNode\*& root, RBNode\* z) {

&#x20;   while (z->parent != nullptr && z->parent->color == RED) {

&#x20;       if (z->parent == z->parent->parent->left) {

&#x20;           RBNode\* uncle = z->parent->parent->right;

&#x20;          &#x20;

&#x20;           if (uncle != nullptr && uncle->color == RED) {

&#x20;               // 情况1：叔叔节点是红色

&#x20;               z->parent->color = BLACK;

&#x20;               uncle->color = BLACK;

&#x20;               z->parent->parent->color = RED;

&#x20;               z = z->parent->parent;

&#x20;           } else {

&#x20;               // 情况2：叔叔节点是黑色

&#x20;               if (z == z->parent->right) {

&#x20;                   // 情况2：z是右孩子，先左旋

&#x20;                   z = z->parent;

&#x20;                   leftRotate(root, z);

&#x20;               }

&#x20;              &#x20;

&#x20;               // 情况3：z是左孩子，进行颜色变换和右旋

&#x20;               z->parent->color = BLACK;

&#x20;               z->parent->parent->color = RED;

&#x20;               rightRotate(root, z->parent->parent);

&#x20;           }

&#x20;       } else {

&#x20;           // 对称情况，父节点在右子树

&#x20;           RBNode\* uncle = z->parent->parent->left;

&#x20;          &#x20;

&#x20;           if (uncle != nullptr && uncle->color == RED) {

&#x20;               // 情况1：叔叔节点是红色

&#x20;               z->parent->color = BLACK;

&#x20;               uncle->color = BLACK;

&#x20;               z->parent->parent->color = RED;

&#x20;               z = z->parent->parent;

&#x20;           } else {

&#x20;               // 情况2：叔叔节点是黑色

&#x20;               if (z == z->parent->left) {

&#x20;                   // 情况2：z是左孩子，先右旋

&#x20;                   z = z->parent;

&#x20;                   rightRotate(root, z);

&#x20;               }

&#x20;              &#x20;

&#x20;               // 情况3：z是右孩子，进行颜色变换和左旋

&#x20;               z->parent->color = BLACK;

&#x20;               z->parent->parent->color = RED;

&#x20;               leftRotate(root, z->parent->parent);

&#x20;           }

&#x20;       }

&#x20;   }

&#x20;   root->color = BLACK;  // 根节点必须为黑色

}
```

#### 4.5.6 插入调整的三种情况

插入调整分为三种情况：

**情况 1**：叔叔节点是红色

- 操作：将父节点和叔叔节点设为黑色，祖父节点设为红色

- 效果：解决了连续红色问题，但祖父节点可能与它的父节点冲突

- 处理：将当前节点上移到祖父节点，继续循环

**情况 2**：叔叔节点是黑色，且当前节点是右孩子（父节点在左）

- 操作：以父节点为支点进行左旋转

- 效果：将情况 2 转换为情况 3

- 注意：旋转后，新的当前节点是父节点，颜色不变

**情况 3**：叔叔节点是黑色，且当前节点是左孩子（父节点在左）

- 操作：将父节点设为黑色，祖父节点设为红色，然后以祖父节点为支点进行右旋转

- 效果：恢复红黑性质，不再有连续红色节点

#### 4.5.7 红黑树的删除操作

红黑树的删除操作更为复杂，需要处理多种情况。删除过程分为：

1. **标准 BST 删除**：找到并删除目标节点

2. **颜色调整**：如果删除的是黑色节点，需要进行复杂的调整

**删除算法实现**：

```Plain Text

&#x20;   RBNode\* y = z;

&#x20;   RBNode\* x;

&#x20;   Color originalColor = y->color;

&#x20;  &#x20;

&#x20;   if (z->left == nullptr) {

&#x20;       x = z->right;

&#x20;       transplant(root, z, z->right);

&#x20;   } else if (z->right == nullptr) {

&#x20;       x = z->left;

&#x20;       transplant(root, z, z->left);

&#x20;   } else {

&#x20;       y = findMin(z->right);

&#x20;       originalColor = y->color;

&#x20;       x = y->right;

&#x20;      &#x20;

&#x20;       if (y->parent == z) {

&#x20;           if (x != nullptr) {

&#x20;               x->parent = y;

&#x20;           }

&#x20;       } else {

&#x20;           transplant(root, y, y->right);

&#x20;           y->right = z->right;

&#x20;           y->right->parent = y;

&#x20;       }

&#x20;      &#x20;

&#x20;       transplant(root, z, y);

&#x20;       y->left = z->left;

&#x20;       y->left->parent = y;

&#x20;       y->color = z->color;

&#x20;   }

&#x20;  &#x20;

&#x20;   if (originalColor == BLACK) {

&#x20;       deleteFixup(root, x);

&#x20;   }

}

&#x20;   while (x != root && (x == nullptr || x->color == BLACK)) {

&#x20;       if (x == x->parent->left) {

&#x20;           RBNode\* w = x->parent->right;

&#x20;          &#x20;

&#x20;           if (w->color == RED) {

&#x20;               // 情况1：兄弟节点是红色

&#x20;               w->color = BLACK;

&#x20;               x->parent->color = RED;

&#x20;               leftRotate(root, x->parent);

&#x20;               w = x->parent->right;

&#x20;           }

&#x20;          &#x20;

&#x20;           if ((w->left == nullptr || w->left->color == BLACK) &&

&#x20;                       w->left->color = BLACK;

&#x20;                   }

&#x20;                   w->color = RED;

&#x20;                   w = x->parent->right;

&#x20;               }

&#x20;              &#x20;

&#x20;               // 情况4：兄弟节点的右孩子是红色

&#x20;               w->color = x->parent->color;

&#x20;               x->parent->color = BLACK;

&#x20;               if (w->right != nullptr) {

&#x20;                   w->right->color = BLACK;

&#x20;               }

&#x20;               leftRotate(root, x->parent);

&#x20;               x = root;

&#x20;           }

&#x20;       } else {

&#x20;           // 对称情况，x是右孩子

&#x20;           RBNode\* w = x->parent->left;

&#x20;          &#x20;

&#x20;           if (w->color == RED) {

&#x20;               // 情况1：兄弟节点是红色

&#x20;               w->color = BLACK;

&#x20;               x->parent->color = RED;

&#x20;               rightRotate(root, x->parent);

&#x20;               w = x->parent->left;

&#x20;          &#x20;

&#x20;                   if (w->right != nullptr) {

&#x20;                       w->right->color = BLACK;

&#x20;                   }

&#x20;              &#x20;

&#x20;               w->color = x->parent->color;

&#x20;               x->parent->color = BLACK;
```

#### 4.5.8 红黑树的复杂度分析

红黑树的时间复杂度：

- 查找、插入、删除操作的时间复杂度均为**O(log n)**

- 树的高度不超过 2log (n+1)，保证了操作的高效性

空间复杂度：

- 每个节点需要额外存储颜色信息（通常 1 位），空间复杂度仍为**O(n)**

#### 4.5.9 AVL 树与红黑树的比较

|特性|AVL 树|红黑树|
|---|---|---|
|平衡严格度|严格平衡（高度差≤1）|大致平衡（最长路径≤2× 最短路径）|
|旋转次数|插入 / 删除时可能多次旋转|插入 / 删除时旋转次数较少|
|查找效率|略高（树更矮）|稳定的 O (log n)|
|插入效率|较低（需频繁旋转）|较高（旋转次数少）|
|适用场景|查询密集型应用|插入 / 删除频繁的应用|
## 第 5 章 B 树与 B + 树

### 5.1 B 树的概念与背景

#### 5.1.1 B 树的发明背景

B 树是由 Rudolf Bayer 和 Edward M. McCreight 于 1970 年在波音研究实验室发明的。当时的背景是，随着计算机存储技术的发展，需要处理越来越大的数据集，而传统的二叉搜索树在处理大规模数据时面临严重的性能问题。

B 树的设计初衷是**解决大规模数据在外部存储设备上高效检索的问题**。它通过多路平衡的方式，显著减少了树的高度，从而降低了访问磁盘的次数。

#### 5.1.2 B 树的定义与性质

B 树是一种自平衡的多路搜索树，具有以下性质：

设 m 为 B 树的阶（degree），则 B 树的每个节点必须满足：

1. 每个节点最多有 m 个子节点

2. 除根节点外，每个节点至少有⌈m/2⌉个子节点

3. 根节点至少有 2 个子节点（除非 B 树只有一个节点）

4. 所有叶子节点都在同一层

5. 节点中的关键字按升序排列

B 树的每个节点结构：

```Plain Text

\[ n, K1, P1, K2, P2, ..., Kn, Pn ]
```

其中：

- n：关键字个数（⌈m/2⌉-1 ≤ n ≤ m-1）

- Ki：第 i 个关键字

- Pi：指向子树的指针

#### 5.1.3 B 树的高度

B 树的高度 h 满足：

```Plain Text

h ≤ log\_{⌈m/2⌉}((N+1)/2) + 1
```

### 5.3 B + 树的原理与应用

#### 5.3.1 B + 树的概念与特点

B + 树是 B 树的一种重要变体，由 Douglas Comer 在 1979 年正式提出。B + 树专门为数据库和文件系统设计，具有以下特点：

1. **所有数据存储在叶子节点**：内部节点只存储关键字，不存储数据

2. **叶子节点形成有序链表**：便于范围查询和顺序访问

3. **关键字可能重复**：内部节点的关键字是其子树最大（或最小）关键字的副本

4. **所有叶子节点在同一层**：保证了查询的稳定性

#### 5.3.2 B + 树与 B 树的对比

|特性|B 树|B + 树|
|---|---|---|
|数据存储位置|所有节点都存储数据|只有叶子节点存储数据|
|内部节点|关键字 + 数据|仅关键字（用于路由）|
|关键字重复|不允许重复|允许重复（内部和叶子）|
|叶子节点链接|无链接|双向链表连接|
|范围查询效率|O(log n + k)|O(log n + k/b)|
|顺序访问|需要中序遍历|直接遍历链表|
