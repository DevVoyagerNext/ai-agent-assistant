# 核心执行协议 (Logic-First Protocol)

## 1. 进入计划模式 (Mandatory Planning)
在进行任何代码或配置变更前，必须通过 `write_file` 在 `docs/plans/` 生成深度技术方案。

### 第一步：生成规划文档 (深度导向)
- **文件名**：`YYYYMMDD_任务简述_v1.md`
- **文档核心要求**：
  1. **核心思路 (Mindset)**：
    - 详细描述技术实现方案及其逻辑闭环。
    - 必须解释“为什么要这样设计”以及“如何处理边界情况/并发/性能风险”。
    - 描述数据在各个组件（Controller, Service, DAO）间的流动逻辑。
  2. **方案详述 (Solution Detail)**：
    - 如果有多种实现方式，必须简述对比结论，并详述最终选定的技术路径。
  3. **详细执行步骤 (Execution Roadmap)**：
    - 严禁模糊描述。必须拆解为 Task 1, Task 2... 每一项都要明确“在哪个文件、改什么、达到什么效果”。
  4. **核心代码要点 (Critical Snippets Only)**：
    - **严禁全量写代码**。仅需展示最核心的：接口定义、关键算法逻辑或复杂的结构体定义（不超过 30 行关键代码）。
  5. **影响文件清单 (Impact Files)**：
    - 列出所有受影响路径及变更类型（Add/Modify/Delete）。

### 第二步：指令阻断与权限确认
- **操作要求**：生成文件后，必须立即停止。
- **交互规范**：
  > [方案已就绪] 详细思路已生成至：docs/plans/xxx.md。
  > 我已在文档中详述了**逻辑实现思路**与**分步执行计划**。
  > 请确认逻辑是否严密。确认后回复“执行”，或提出您的修改建议。

## 2. 执行阶段 (Execution)
- 严禁跳跃执行。必须按照计划中的 Roadmap 逐项实施。
- 每一个 Task 完成后，需在终端同步进度。
