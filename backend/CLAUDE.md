# 核心执行协议 (Plan-First Protocol)

## 1. 进入计划模式 (Mandatory Planning)
在执行任何代码变更（不论大小）前，必须通过 `write_file` 在 `docs/plans/` 生成一份任务规划。

### 第一步：生成规划文档模板
- **文件名**：`YYYYMMDD_TASK_PLAN.md`
- **深度内容要求**：
  1. **现状分析**：分析当前代码结构及登录/工具链逻辑。
  2. **修改步骤 (TODO List)**：将任务拆解为具体的子任务（Task 1, Task 2...）。
  3. **核心代码设计**：展示关键函数/接口的完整源码（严禁伪代码）。
  4. **影响文件列表**：清晰列出所有将变动的文件。

### 第二步：计划审核与编辑权
- **操作要求**：生成文件后，必须立即停止。
- **交互规范**：
  > [Plan 已生成] 路径：docs/plans/xxx.md。
  > 您可以：
  > 1. 回复“执行”：我将严格按照计划顺序开始。
  > 2. 回复“修改”：告诉我哪里需要调整，我会更新计划。
  > 3. 手动编辑：您可以直接在 IDE 中打开该文件修改内容，保存后告诉我“按新计划执行”。

## 2. 执行阶段 (Execution)
- 只有在获得明确授权后，方可开始调用 `edit_file`。
- **顺序执行**：必须按照计划中的 Task 1, Task 2 顺序执行，每完成一个大步骤建议询问一次进度。
